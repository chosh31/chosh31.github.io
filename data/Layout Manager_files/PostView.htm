13.3.1 Layout Manager란

Container 계열의 컴포넌트들은 다른 컴포넌트들을 포함할 수 있다. 컴포넌트를 포함할 때 어떻게 배치할 것인지에 대한 방법을 레이아웃(Layout)이라고 한다.

□ 레이아웃(Layout)
◇ Container 계열의 컴포넌트가 다른 컴포넌트를 포함할 때 어떻게 배치할 것인지에 대한 방법을 레이아웃(Layout)이라고 한다.

보통 레이아웃을 지정하는 클래스를 레이아웃 관리자(배치관리자)라고 하며 다음과 같이 5가지 종류가 있다.

□ 레이아웃 관리자(Layout Manager)
◇ FlowLayout
◇ BorderLayout
◇ GridLayout
◇ CardLayout
◇ GridBagLayout

FlowLayout 은 삽입되는 순서대로 보이는 레이아웃을 말하며, BorderLayout은 동.서.남.북.중앙의 다섯 곳의 위치에 삽입할 수 있는 레이아웃이다. 그리고 GridLayout은 Grid 형식으로 분할한 뒤 분할된 곳에 순서대로 컴포넌트를 삽입하게 된다. CardLayout은 카드처럼 컴포넌트를 겹쳐서 배치하며, 제일 앞에 있는 컴포넌트만 보이게 하는 방식이다.

□ FlowLayout
◇ 삽입된 순서대로 보이는 레이아웃

□ BorderLayout
◇ 동.서.남.북.중앙의 다섯 곳의 위치에 삽입할 수 있는 레이아웃

□ GridLayout
◇ Grid 형식으로 분할한 뒤 분할된 곳에 순서대로 컴포넌트를 삽입하는 레이아웃

□ CardLayout
◇ 카드처럼 컴포넌트를 겹쳐서 배치하며, 제일 앞에 있는 컴포넌트만 보이게 하는 방식

GridBagLayout 은 Grid 형식으로 분할하는 측면에서 본다면 GridLayout과 비슷하다. GridLayout은 분할된 한 칸에 하나의 컴포넌트만 삽입할 수 있지만, GridBagLayout은 여러 칸에 하나의 컴포넌트를 삽입하는 방식이다. 복잡한 화면을 디자인할 때에는 GridBagLayout을 많이 사용한다. 먼저 FlowLayout부터 알아보자.
